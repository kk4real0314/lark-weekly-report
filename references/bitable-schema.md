# 多维表格结构定义

使用本 skill 前必须确保 bitable 包含以下 5 张核心表 + 1 张台账表，每张表的必需字段（名称+类型）如下。

> 字段名通过 `weekly-report-config.json` 的 `fields` 映射配置，不要求与下方示例名完全一致，但**类型和用途必须匹配**。

## 1. 项目管理表

用途：项目主档，所有子表通过双向关联指回此表。

| 必需字段 | 类型 | 用途 |
|---|---|---|
| 项目名称 | 文本 | 主字段，查询和展示用 |
| 项目状态 | 单选 | 活跃/非活跃判断 |

可选但推荐：区域/类型(单选)、关注情况(单选)、项目经理(人员)、交付负责人(人员)、合同金额(数字)、风险描述(文本)

## 2. 里程碑管理表

用途：验收/回款节点。

| 必需字段 | 类型 | 用途 |
|---|---|---|
| 里程碑状态 | 单选 | 进行中/已关闭/已取消/待评审 |
| 付款类型 | 单选 | 首付款/验收款/维保款等 |
| 计划验收金额 | 数字 | 万 |
| 当前计划时间 | 日期 | 筛选窗口用 |
| 所属项目 | 双向关联 | → 项目管理表 |

可选：付款比例(数字)、实际达成时间(日期)、偏离天数(公式)、交付负责人(人员)

## 3. 任务列表表

用途：任务执行跟踪。

| 必需字段 | 类型 | 用途 |
|---|---|---|
| 任务名称 | 文本 | 主字段 |
| 任务状态 | 单选 | 已完成/进行中/未开始等 |
| 开始时间 | 日期 | 时间窗口筛选 |
| 计划完成时间 | 日期 | 时间窗口筛选 |
| 所属项目 | 双向关联 | → 项目管理表 |

可选：任务类型(单选)、优先级(单选)、进度百分比(数字)、负责人(人员)、实际完成时间(日期)

## 4. 问题跟踪表

用途：问题/BUG 管理。

| 必需字段 | 类型 | 用途 |
|---|---|---|
| 问题描述 | 文本 | 主字段 |
| 解决状态 | 单选 | 已解决/未解决/验证中/未开始/搁置 |
| 问题级别 | 单选 | P0/P1 |
| 问题类型 | 单选 | BUG/功能咨询/实施等 |
| 问题产生时间 | 日期 | 时间窗口筛选 |
| 所属项目 | 双向关联 | → 项目管理表 |

可选：所属产品(多选)、研发负责人(人员)、解决时间(日期)

## 5. 变更记录表

用途：变更全流程管理。

| 必需字段 | 类型 | 用途 |
|---|---|---|
| 变更内容 | 文本 | 主字段 |
| 变更状态 | 单选 | 变更完成/待审核/未开始/回滚/已取消 |
| 变更级别 | 单选 | 重大变更/一般变更/正常变更 |
| 变更类型 | 单选 | 解决BUG/功能优化/定制化开发等 |
| 变更开始时间 | 日期 | 时间窗口筛选 |
| 所属项目 | 双向关联 | → 项目管理表 |

可选：涉及功能模块(多选)、影响范围(文本)、变更方案(文本)、研发负责人(人员)

## 6. 周报台账表

用途：记录每次生成的周报元数据。

| 必需字段 | 类型 | 用途 |
|---|---|---|
| 周报标题 | 文本 | 标识 |
| 报告状态 | 单选 | 草稿/已发布 |
| 周报文档 | 超链接 | 指向飞书文档 |

可选：汇报日期(日期)、周期开始(日期)、周期结束(日期)、各类统计数字字段

## 关联关系

```
项目管理 ←→ 里程碑管理（所属项目）
项目管理 ←→ 任务列表（所属项目）
项目管理 ←→ 问题跟踪（所属项目）
项目管理 ←→ 变更记录（所属项目）
```

所有子表通过 `所属项目` 双向关联指回项目管理表。这是数据聚合的基础。
