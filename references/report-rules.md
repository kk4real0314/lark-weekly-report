# 周报生成规则

> 本文件定义项目分析周报的完整生成规范。SKILL.md 通过引用本文件，生成周报时**必须先读本文件**。

## 报告模板

### 项目周报（7 章结构）

```markdown
# 项目分析周报
> 汇报周期：{上周四 yyyy/MM/dd} ~ {本周三 yyyy/MM/dd}

---

## 一、项目分析
项目总览 + 有动态项目列表 + 本章小结

## 二、里程碑分析
里程碑总览 + 甘特图 + 明细表 + 本章小结

## 三、本周任务
任务总览 + 明细表（含起止列）+ 本章小结

## 四、下周任务
下周计划表 + 重点工作方向（有序列表）+ 本章小结

## 五、问题分析
问题总览 + P0 表格 + P1 表格 + 本章小结

## 六、变更分析
变更总览 + 明细表 + 本章小结

## 七、汇报总结
有序列表（6~8 条要点）
```

## 数据查询范围

> **时间窗口锚点 = 周期本身**，不使用"过去 N 天"这种相对表述。

### 周期计算

```
today = 当前日期
this_wed = today 向前找到最近的周三（含今天）
last_thu = this_wed - 6 天
周期 = last_thu ~ this_wed
```

用户指定特殊周期时按用户要求，但数据窗口始终与周期对齐。

### 数据窗口

| 数据类型 | 时间窗口 | 筛选字段 |
|---|---|---|
| 任务（本周） | `开始时间` 或 `计划完成时间` 在周期内 | 开始时间 / 计划完成时间（OR） |
| 任务（下周） | `开始时间` 或 `计划完成时间` 在周期结束后 7 天内 | 开始时间 / 计划完成时间 |
| 问题 | `问题产生时间` 在周期内 | 问题产生时间 |
| 变更 | `变更开始时间` 在周期内 | 变更开始时间 |
| 里程碑 | `当前计划时间` 在本月1日 ~ 未来50天；`实际达成时间` 在周期内 | 当前计划时间 / 实际达成时间 |
| 逾期项 | `计划完成时间` 在周期内且未完成 | 计划完成时间 |

> **时间范围是红线，必须严格遵守**：
> 1. **任务不使用`实际完成时间`筛选**——只用`开始时间`和`计划完成时间`，否则会拉入计划期在数月前、仅本周关闭的历史任务
> 2. **问题必须按`问题产生时间`在周期内筛选**——不查全量存量（解决状态≠已解决），否则拉入数月前历史问题
> 3. **所有数据都必须受时间窗口约束**，无一例外。窗口外的数据无论状态如何都不纳入周报

**关键约束**：不查询"所有逾期"（`计划完成时间 < 今天`），只查周期内到期的。历史长期逾期不纳入周报。

## 周报生成约束

- **项目名称用全称**：报告里所有项目名必须使用多维表格中的完整名称，不用简称
- **项目列表只展示有动态的项目**：项目分析章节只展示周期内有任务/问题/变更/里程碑变化的项目
- **里程碑展望窗口 50 天**：里程碑分析的时间窗口为未来 50 天，避免月末/季末边界遗漏
- **活跃项目定义**：项目状态在 config.enums.inactive_statuses 中的项目不纳入周报
- **搁置问题不展示**：解决状态=搁置的问题不在周报中展示，P0/P1 表格中也不出现
- **P0/P1 问题均用表格展示**：6 列表（状态/类型/问题描述/所属产品/产生时间/研发负责人）
- **任务表含起止时间列**：任务清单表格必须包含"起止"列（开始时间~计划完成时间）
- **下周重点工作方向用有序列表**：编号列表，不依赖表格
- **汇报总结用有序列表**：编号列表，不用段落
- **不展示工时/成本数据**：报告中不出现工作量、人天、成本等内部数据
- **逾期项仅限周期内**：只展示计划完成时间在周期内的逾期任务
- **周报不写内部规则**：数据来源/筛选口径不在文档中出现
- **标题周期规则**：周报周期为上周四到本周三；用户指定特殊周期时按用户要求

## 生成规则

1. **一/项目分析**：列出活跃项目中有动态的项目，附状态/区域/关注情况
2. **二/里程碑分析**：
   - `里程碑实际达成时间` 在周期内 → 已达成
   - `里程碑状态=进行中` 且 `当前计划时间` 在里程碑窗口内 → 进行中
   - `偏离天数 > 0` → 延期
3. **三/本周任务**：按项目维度展开，标注状态和进度
4. **四/下周任务**：到期任务表 + 重点工作方向
5. **五/问题分析**：P0/P1 分别用表格，不展示搁置项
6. **六/变更分析**：变更明细表 + 风险提示
7. **七/汇报总结**：6~8 条要点，有序列表

## 生成流程

1. **前置检查**：扫描超期未更新的任务，提醒用户补数据后再生成
2. **读取配置**：从工作区 `weekly-report-config.json` 读取
3. **计算时间窗口**：上周四 ~ 本周三，里程碑窗口 = 本月1日 ~ 未来50天
4. **拉数据**：按数据查询范围从 5 张核心表拉取（项目 + 里程碑 + 任务 + 问题 + 变更）
5. **聚合**：按项目维度聚合，识别活跃项目集合，过滤非活跃项目
6. **组装**：按 7 章模板填充，遵守全部生成约束
7. **生成图表**：matplotlib 生成 3 张图（项目状态饼图、里程碑甘特图、问题分布图）
8. **写入飞书文档**：输出为 docx 到 config.output.folder_token 指定的文件夹
9. **写台账**（可选）：若 config 中 ledger 表已配置，在台账表新增一行，记录标题、日期、doc_id、状态=草稿
10. **完成**：返回文档链接给用户

## 已知坑点

| 坑 | 说明 |
|---|---|
| 枚举值前后空格 | 部分枚举值有尾部空格，筛选时必须完整匹配 |
| 关联字段写入格式 | 必须 `{"record_ids":[...],"table_id":"..."}`，不是字符串数组 |
| 日期字段格式 | 全部毫秒级 Unix 时间戳（13 位整数），不是字符串 |
| 进度百分比 | 0-1 而不是 0-100 |
| replace_range 不可靠 | 飞书文档 `replace_range` 对表格内内容不可靠；优先用 `block_insert_after` + `block_replace` |
| 表格列宽 | 飞书文档 API 不支持修改 `column_width`，需在飞书 UI 手动调整 |
| 里程碑数据源 | 只用里程碑管理表，不混入任务表的"任务类型=里程碑" |
| 问题查询逻辑 | 问题必须按`问题产生时间`在周期内筛选，不查全量存量 |
| 里程碑查询必须加日期上限 | 不能只按 `状态=进行中` 查，必须加 `当前计划时间` 在本月~未来50天 内 |
| 任务查询必须按日期字段筛选 | 不能按"所属项目"反查；不使用实际完成时间筛选 |
| media-insert 只 append 到末尾 | `docs +media-insert` 只能在文档末尾插入，插入后必须 `block_move_after` 移到正确位置，从后往前移 |
| 特殊周期与里程碑规则 | 用户指定特殊日期范围时，仅适用于任务/问题/变更；里程碑始终按规则走（本月+未来50天） |
| 时间轴图只画窗口内数据 | 只画查询窗口内的里程碑；图幅要够大（宽≥16in 高≥6in DPI≥200） |
